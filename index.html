Travel Expense Report Application

This application is a web-based tool designed to generate custom Excel reports based on user-submitted data. Built using the Flask framework, the application provides an intuitive interface for users to input various data points, which are then used to populate a predefined Excel template. The application runs within a Docker container on an Oracle Compute instance and is served by the Gunicorn production server. An Nginx server, set up inside the Oracle instance but outside the Docker container, acts as an HTTPS termination endpoint and a proxy to the Docker containers. This setup ensures secure and efficient handling of requests.

The application also features a robust CI/CD pipeline for seamless deployment from development to production. When changes are pushed to the GitHub repository, a new Docker image is automatically built and pushed to Docker Hub. The Oracle instance then pulls the latest image, replacing the running container with the updated version. This workflow ensures that the application remains up-to-date with minimal manual intervention, providing a reliable and streamlined experience for both users and developers.

Chattanooga-Hackathon-2024

I have built a comprehensive registration website for a upcoming hackathon event to be held in Chattanooga, TN. The website, hosted on AWS S3, provides a seamless user experience for participants to register for the event. Leveraging modern web technologies and design principles, the site features an intuitive registration form that allows users to sign up for the hackathon effortlessly. The static website, which is publicly accessible, showcases event details, schedules, and sponsor information, creating a vibrant and informative landing page for potential attendees.

The AWS infrastructure supporting this registration site is both robust and secure. User email signups are managed by an AWS Lambda function sitting behind an API Gateway, ensuring scalability and reliability. This Lambda function performs CRUD operations on a list of user emails stored in a separate, secure S3 bucket, maintaining data privacy and integrity. I also implemented a full CI/CD pipeline for streamlined development and deployment processes. This setup ensures that updates to the site and backend services can be deployed efficiently and securely.

Devops and Cloud Engineering Capstone

Our capstone project for the Code Platoon DevOps and Cloud Engineering bootcamp involved building a scalable and resilient cloud infrastructure on AWS Elastic Kubernetes Service (EKS) for a Django/React/PostgreSQL application. The application architecture includes a Virtual Private Cloud (VPC) segmented into subnets across different Availability Zones (AZs), ensuring high availability and fault tolerance. An Application Load Balancer (ALB) distributes incoming traffic, while a Network Load Balancer (NLB) handles backend services. The deployment strategy involves building Docker images for the application components, pushing them to Docker Hub, and deploying them to the EKS cluster. Kubernetes manages the pods running the React frontend, Django backend, and PostgreSQL database. AWS services, such as Elastic Load Balancers, ensure efficient traffic management and high availability. The AWS resources are provisioned using Terraform, enabling infrastructure as code for a consistent and repeatable setup. EKS simplifies Kubernetes management, providing a secure and integrated environment with AWS logging, monitoring, and security services. This project demonstrates our ability to build and manage a comprehensive cloud infrastructure, showcasing the skills acquired during the bootcamp.
